Network Isolation Rules

4 rules · Applied to: All environments
BLOCKS DEPLOYNo public databases
Database resources (RDS, DynamoDB, ElastiCache, OpenSearch) must never be placed in a public subnet. The platform auto-redirects to private subnet and shows a warning.
IF resource.type IN ["rds", "dynamodb", "elasticache", "opensearch"]
THEN resource.subnet.type MUST BE "private"
ACTION: auto_move_to_private + notify_user
📎 All envs👤 admin☁️ AWS + Azure
BLOCKS DEPLOYEncryption at rest required
All storage resources (S3, EBS, RDS, EFS) must have encryption enabled. Deployment blocked if disabled.
IF resource.type IN ["s3", "ebs", "rds", "efs"]
THEN resource.encryption MUST BE true
ACTION: block_deploy + show_validation_error
📎 acc, prd👤 admin☁️ AWS + Azure
WARNINGProduction requires Multi-AZ
Production stateful resources should be Multi-AZ. Warns on canvas, blocks during deploy review.
IF env.type == "production" AND resource.stateful == true
THEN resource.multi_az MUST BE true
ACTION: warn_on_canvas + block_deploy_review
📎 prd only👤 admin☁️ AWS
INFOVPC CIDR range constraints
VPCs must use pre-approved CIDR ranges. Cross-env peering must not overlap. Platform auto-assigns non-conflicting ranges.
IF resource.type == "vpc"
THEN resource.cidr IN approved_cidr_pool
AND NO overlap WITH other_envs.vpc.cidr
ACTION: auto_assign_cidr
📎 All envs👤 admin☁️ AWS + Azure
